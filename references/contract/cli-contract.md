# CLI Contract (T0.10)

> **Verified live on 2026-04-26** (post-T0.13 baseline). This contract is what
> Iteration 6 (modular refactor) MUST preserve byte-identical for read-only
> invocations. Any intentional change requires snapshot regen in same commit.

## 1. Top-level invocations

| Invocation | Exit | stdout | stderr | Side effects |
|---|---|---|---|---|
| `python cli.py --list` | 0 | JSON array (pretty, indent=2) | empty | none |
| `python cli.py --describe <tool>` | 0 (valid) / 1 (unknown tool) | JSON object (pretty, indent=2) | error message on unknown tool | none |
| `python cli.py --describe` (no arg) | 2 | empty | argparse usage error | none |
| `python cli.py --call '{...}'` | 0 ok / 1 error | JSON result | error message | varies (read=none; write=API mutation; **also currently writes config.json via auto-refresh — fixed in T3.9**) |
| `python cli.py --refresh-cache` | 0 / 1 on error | progress to stderr | progress | writes `references/*.json`, writes `projects_cache.md`, **currently rewrites `config.json` — fixed in T3.9** |
| `python cli.py --doctor` | 0 (all pass) / 1 (some fail) / 2 (config missing) | JSON status | empty | NONE (read-only invariant; verified by T0.13 test) |
| `python cli.py --help` | 0 | argparse help text | empty | none |
| `python cli.py` (no args) | 0 | argparse help text (via `parser.print_help()`) | empty | none |

### Notes on argparse semantics
- `--describe` without an argument: argparse rejects, prints usage to stderr,
  exits 2. This is standard argparse behaviour, **not** a tool error.
- `python cli.py` (no args): `parser.print_help()` is invoked and the function
  returns; default exit 0. (Distinct from "missing required arg" which would be 2.)
- Unknown `--describe <tool>`: tool prints `Tool not found: <name>` to stderr,
  exits 1.

- `--doctor` — live HTTP responses; checked structurally, not byte-identical.
- `--call` — depends on live API state.
- `--refresh-cache` — has live side effects.

## 4. Argument naming invariants

- Argument names and required-flags are stable across MINOR/PATCH versions.
- MAJOR version bump required for any rename or removal of a parameter.
- Adding new optional parameters is a MINOR bump.

## 5. Output format invariants

- All JSON outputs use `json.dumps(..., indent=2, ensure_ascii=False)`.
- Key order = Python dict insertion order (Python 3.7+ guarantee). NOT
  alphabetical. The order is determined by:
  - For `--list`: order of `TOOL_CATALOG` items.
  - For `--describe`: schema construction order (name, description, inputSchema).
- UTF-8 encoding without BOM.
- Final newline (`print` appends `\n`).

## 6. Exit codes

| Code | Meaning |
|---|---|
| 0 | Success |
| 1 | Application error (CRUD failed, schema invalid, doctor checks failed, unknown tool, malformed JSON in `--call`) |
| 2 | Usage error (argparse rejection, config.json missing/invalid) |

## 7. Forbidden behaviour (regression check list)

These constitute parity failures even if snapshot text is unchanged:

1. Any side effect from `--list`, `--describe`, `--doctor`, `--help`, or `python cli.py` (no args).
2. `--doctor` writing to `config.json`, `references/*.json`, `projects_cache.md`, or any file.
3. `--describe <tool>` returning a schema that doesn't validate as JSON Schema draft-07
   (currently fails this — fixed in Iteration 4 / T4.3).
4. `tools.json` count diverging from `--list` count (currently 60 vs 63 — fixed in T4.5).
