# Джедайские техники Дорофеева для Singularity

Скилл продуктивности, основанный на книге Максима Дорофеева "Джедайские техники". Реализует 8 режимов управления задачами через Singularity App.

## Принцип активации

**Режимы активируются ТОЛЬКО по запросу пользователя.** Когда фразы пользователя совпадают с триггерами в frontmatter — активировать соответствующий режим. Никогда не навязывать режимы, не предлагать их без запроса, не запускать автоматически.

## Зависимость от скилла singularity

Все операции с Singularity выполняются через скилл `singularity` (63 инструмента REST API). Путь к CLI: `~/.claude/skills/singularity/cli.py`. Формат вызова:

```bash
python ~/.claude/skills/singularity/cli.py --call '{"tool":"tool_name","arguments":{...}}'
```

Перед любой работой убедиться, что кэш `references/projects.json` и `references/tags.json` существует. Если нет — вызвать `rebuild_references`.

## Базовые принципы Дорофеева (держать в голове при всех режимах)

| Принцип | Суть |
|---------|------|
| **Мыслетопливо** | Ограниченный ресурс. Каждое "я запомню" — утечка. Сложное — на утро |
| **Обезьянка** | Внутренний исполнитель. Любит простые, конкретные задачи с глаголом. Боится мутного |
| **4 критерия задачи** | 1) Глагол 2) "Что конкретно?" 3) Минимум мышления 4) Первоочередное действие |
| **>30 мин = проект** | Если задача больше 30 минут — это проект, нужна разбивка |
| **WIP-лимит** | Не больше 3-5 задач "в работе". Чтобы закончить раньше — начни позже |

## Инициализация (ленивая, по первому использованию)

При первом использовании соответствующего режима создать необходимые сущности в Singularity. НЕ создавать всё сразу — только то, что нужно для текущего режима.

### Проекты инкубатора (для Режима 5)

Создавать при первом вызове Режима 5. Перед созданием проверить наличие через кэш `references/projects.json`.

| Проект | Emoji | reviewValidationInterval | Назначение |
|--------|-------|--------------------------|------------|
| Инкубатор: Шкаф | `1f3e0` | 7 | Еженедельный пересмотр. Идеи "на этой неделе не до этого" |
| Инкубатор: Балкон | `26f0` | 30 | Ежемесячный пересмотр. Идеи "может через пару месяцев" |
| Инкубатор: Дача | `1f3d6` | 90 | Ежеквартальный пересмотр. Идеи "когда-нибудь/может быть" |

```bash
python cli.py --call '{"tool":"project_create","arguments":{"title":"Инкубатор: Шкаф","emoji":"1f3e0","reviewValidationInterval":7}}'
python cli.py --call '{"tool":"project_create","arguments":{"title":"Инкубатор: Балкон","emoji":"26f0","reviewValidationInterval":30}}'
```

## Режим 1: Разгрузка

**Слоган:** "Из головы — в систему"

### Когда активировать

Пользователь говорит: "разгрузи голову", "запиши что в голове", "у меня куча мыслей", "мне надо всё записать", "выгрузи из головы", "dump", "что в голове". Или просто начинает перечислять кучу разрозненных мыслей/задач/идей потоком сознания.

### Алгоритм

1. **Принять поток сознания.** Попросить пользователя говорить свободным текстом — всё, что крутится в голове. Не прерывать, не уточнять в процессе. Просто собрать всё.

2. **Классифицировать каждый элемент** и обработать по таблице:

| Тип элемента | Признаки | Действие в Singularity |
|--------------|----------|----------------------|
| Конкретная задача (<30 мин) | Понятно что делать, один шаг | `task_create` — переформулировать по 4 критериям, назначить `start: today`, `projectId` по контексту |
| Большое дело (>30 мин) | Несколько шагов, неясен план | `project_create` + первая конкретная задача-"вход в проект" |
| Справочная информация | Факт, телефон, адрес, ссылка | `task_create` с `isNote: true` в блокноте "Справочные материалы" |
| Встреча/событие | Привязка ко времени | `task_create` с `start` (ISO datetime GMT+3), `useTime: true`, `notifies: [60, 15]` |
| Идея "на потом" | "хорошо бы", "когда-нибудь", "может быть" | В Инкубатор (Режим 5): задача в проекте "Инкубатор: Шкаф", `deferred: true`, тег `incubator` |
| Мутное/непонятное | Не ясно что делать | Переформулировать, задав вопрос "Чтобы что?" или "Какой первый шаг?" |

3. **Переформулировать по 4 критериям Дорофеева:**

**Правила переформулировки:**

| # | Критерий | Пример нарушения | Исправление |
|---|----------|------------------|-------------|
| 1 | Начинается с глагола действия | "Стоматолог" | "Позвонить в поликлинику и записаться к стоматологу" |
| 2 | Отвечает на "Что конкретно сделать?" | "Разобраться с отчётом" | "Открыть шаблон отчёта и заполнить раздел 1" |
| 3 | Минимум мышления (проще сделать, чем записать) | "Улучшить презентацию" | "Заменить скриншоты на слайдах 3-5 на актуальные" |
| 4 | Первоочередное действие (не конечная цель) | "Вылечить зуб" | "Позвонить в поликлинику и записаться к стоматологу" |

**Глаголы-помощники:** Позвонить, Написать, Отправить, Найти, Открыть, Создать, Купить, Спросить, Проверить, Скачать, Записаться, Выделить N минут.

**Типичные переформулировки:**

| Пользователь говорит | Переформулировать в |
|---------------------|-------------------|
| "встреча с X" | "Написать X и предложить время встречи" |
| "разобраться с Y" | "Открыть Y и определить первый конкретный шаг" |
| "проконтролировать Z" | "Написать Z и спросить, как продвигается задача" |
| "надо бы [что-то]" | "Выделить 15 минут и [конкретное действие]" |
| "узнать про X" | "Загуглить X" / "Спросить у [кого] про X" |
| "подумать о Y" | "Выделить 15 минут: записать 3 варианта решения Y" |
| "не забыть Z" | "Написать Z до конца дня" / конкретное действие |
| "стоматолог" | "Позвонить в поликлинику и записаться к стоматологу" |
| "отчёт" | "Открыть шаблон отчёта и заполнить первый раздел" |

4. **Показать сводку** в конце:

```
Разгрузка завершена:
- Задачи: 7 (из них 2 на сегодня)
- Проекты: 2 (с задачами-входами)
- Заметки: 1
- Встречи: 1
- В инкубатор: 3
```

### Поведение при ошибках

- Если проект для задачи не определён — создать задачу в Inbox (без `projectId`), добавить тег `jedi-inbox`
- Если пользователь перечислил <3 элемента — всё равно обработать, не требовать "ещё"
- Если элемент дублирует существующую задачу — предупредить, спросить "Обновить существующую или создать новую?"

## Режим 2: Обзор

**Слоган:** "15 минут, которые экономят неделю"

### Когда активировать

Пользователь говорит: "обзор задач", "пересмотреть задачи", "еженедельный обзор", "недельный обзор", "вечерний обзор", "ежедневный обзор", "что с моими делами", "навести порядок в задачах".

### Три варианта

Определить вариант по контексту. Если неясно — по умолчанию **Еженедельный обзор**.

---

### A. Еженедельный обзор

Полный пересмотр всей системы задач. Рекомендуется раз в неделю (воскресенье вечер или понедельник утро).

**Шаги:**

**1. Загрузить все незавершённые задачи:**
```bash
python cli.py --call '{"tool":"task_list","arguments":{"max_count":1000}}'
```
Отфильтровать на клиенте: `checked == 0` (незавершённые).

**2. Найти "протухшие" задачи:**
Задачи, где `modificatedDate` старше 3+ дней и `checked == 0`. Это признак "баннерной слепоты" — задача стала невидимой.

По каждой протухшей:
- Показать текущую формулировку
- Спросить: "Ещё актуально? Если да — переформулируем?"
- Если актуально — переформулировать по 4 критериям, `task_update` с новым `title` и `start: today`
- Если неактуально — `task_update` с `checked: 2` (cancelled)

**3. Проверить отложенные:**
Задачи с `deferred: true`. По каждой: "Это всё ещё актуально? Активировать или оставить?"
- Активировать: `task_update` с `deferred: false`, `start: today`
- Оставить: без изменений
- Удалить: `task_update` с `checked: 2`

**4. Проверить проекты:**
Загрузить список проектов из кэша `references/projects.json`. Для каждого проекта — загрузить задачи через `task_list` с `project_id`. Найти проекты, где все задачи `checked != 0` (нет активных).

По каждому такому проекту: "Проект [название] — все задачи завершены. Проект завершён или нужен следующий шаг?"
- Завершён: `project_update` с `archived: true`
- Нужен шаг: создать новую задачу через `task_create`

**5. Техника "Чтобы что?":**
По каждой застрявшей задаче (протухшая, непонятная, мутная) — задать вопрос: "Зачем тебе эта задача? Какую цель она приближает?" Если пользователь не может ответить за 10 секунд — задача кандидат на удаление или инкубатор.

**6. "Волшебная фея":**
В конце обзора: "Если бы прямо сейчас было 20 свободных минут — что бы ты сделал из всего списка?"
Выбранное пометить как pinned: `task_update` с `state: 0`.

**6.5. Баланс-чек (Кислородная маска):**
Проверить: есть ли среди задач на неделю хотя бы 1 личная (Здоровье, Отдых, Хобби, Семья). Если нет — предложить добавить (см. Режим 7, раздел "Баланс-чек").

**7. Обновить reviewValidationDate на проектах** (если используются проекты с `reviewValidationInterval`):
```bash
python cli.py --call '{"tool":"project_update","arguments":{"id":"P-...","reviewValidationDate":"2026-03-05"}}'
```

---

### B. Ежедневный (вечерний) обзор

Быстрый пересмотр задач на сегодня и подготовка завтрашнего дня. 5-10 минут.

**Шаги:**

**1. Показать задачи на сегодня:**
```bash
python cli.py --call '{"tool":"task_list","arguments":{"start_date_from":"TODAY","start_date_to":"TODAY","max_count":100}}'
```
**Подставлять актуальную дату** в формате `YYYY-MM-DD` (например, `2026-02-26`). Никогда не хардкодить дату.

**2. Завершённые (checked == 1):** По каждой — "Какой следующий шаг?" Если есть — создать задачу на завтра.

**3. Незавершённые (checked == 0):** По каждой:
- Перенести на завтра: `task_update` с `start: завтра`
- Переформулировать (если мутная)
- Отменить: `task_update` с `checked: 2`

**4. Задачи на завтра:** Показать список. По каждой: "Всё ли понятно обезьянке?" Если формулировка мутная — переформулировать по 4 критериям.

**5. Связь с Режимом 8:** После вечернего обзора предложить заполнить "Итоги дня" (Режим 8).

---

### C. Спонтанный обзор

Мини-обзор на 5-10 минут. Для ситуаций "есть 5 минут в очереди", "что посмотреть быстренько".

**Алгоритм:**
1. Загрузить все незавершённые задачи через `task_list`
2. Выбрать 3-5 задач для ревью. Стратегия: 1-2 самые старые по `modificatedDate`, 1-2 случайных из середины, 1 самая свежая. Это обеспечивает покрытие и свежих, и забытых задач.
3. По каждой задать 5 вопросов:

| # | Вопрос | Если "да" |
|---|--------|-----------|
| 1 | Формулировка ещё понятна? | Оставить |
| 2 | Ещё актуально? | Оставить |
| 3 | Можно конкретнее? | `task_update` с новым `title` |
| 4 | Можно вообще не делать? | `task_update` с `checked: 2` |
| 5 | Можно объединить с другой? | Объединить, удалить дубль |

4. Показать итог: "Обзор 5 задач: 2 переформулированы, 1 удалена, 2 без изменений"

## Режим 3: Разбивка (Декомпозиция)

**Слоган:** "Слона — по кусочкам"

### Когда активировать

Пользователь говорит: "разбей на задачи", "это слишком большое", "не знаю с чего начать", "как съесть слона", "декомпозируй", "разбей проект". Также предлагать автоматически в Режиме 1 (Разгрузка), если элемент оценивается >30 минут.

### Алгоритм

**1. Принять вводные:**
- Если пользователь дал ID задачи или URL — загрузить через `task_get` / `task_full`
- Если описание текстом — работать с текстом

**2. Определить тип:**

| Тип | Признаки | Подход |
|-----|----------|--------|
| **Проект** | Известен результат, можно спланировать шаги | Полная декомпозиция (шаг 3) |
| **Дело/Case** | Известен результат, план неясен | Только 1-2 следующих шага (шаг 4) |
| **Образ жизни** | Нет конца, привычка | Привычка + стартовая задача (шаг 5) |

Если тип неясен — спросить: "Ты знаешь, какой результат хочешь получить? Можешь представить план целиком?"

**3. Для Проекта (известен план):**

a) Определить **Минимально приемлемый результат (MVP):** "Что сделать, если проект закроется послезавтра? Что самое минимальное, но законченное?"

b) Создать проект:
```bash
python cli.py --call '{"tool":"project_create","arguments":{"title":"Название проекта","emoji":"подходящий_emoji"}}'
```

c) Если >5 задач — создать секции через `task_group_create`:
```bash
python cli.py --call '{"tool":"task_group_create","arguments":{"title":"Название секции","parent":"P-project-uuid"}}'
```

d) Создать задачи. **Каждая задача:**
- <30 минут на выполнение
- Начинается с глагола
- Первая задача с `start: today` (точка входа)
- Остальные без даты (или с логичной последовательностью дат)

```bash
python cli.py --call '{"tool":"task_create","arguments":{"title":"Глагол + конкретное действие","projectId":"P-...","start":"2026-02-26"}}'
```

e) К сложным шагам — чеклист:
```bash
python cli.py --call '{"tool":"checklist_create","arguments":{"parent":"T-task-uuid","title":"Подшаг 1"}}'
python cli.py --call '{"tool":"checklist_create","arguments":{"parent":"T-task-uuid","title":"Подшаг 2"}}'
```

**4. Для Дела/Case (план неясен):**

Принцип "рационального фланёра": маршрут пересматривается на каждом шагу. Не пытаться спланировать всё.

a) Создать проект:
```bash
python cli.py --call '{"tool":"project_create","arguments":{"title":"Название дела","emoji":"подходящий_emoji"}}'
```

b) Записать "Видение результата" в заметку проекта:
```bash
python cli.py --call '{"tool":"note_create","arguments":{"containerId":"P-project-uuid","content":[{"insert":"Видение результата: [описание]\n"}],"contentType":"delta"}}'
```

c) Создать ТОЛЬКО 1-2 следующих конкретных шага. НЕ весь план:
```bash
python cli.py --call '{"tool":"task_create","arguments":{"title":"Первый конкретный шаг","projectId":"P-...","start":"2026-02-26"}}'
```

d) В описании первой задачи или в заметке проекта: "После выполнения — определить следующий шаг."

**5. Для Образа жизни (привычка):**

a) Создать привычку:
```bash
python cli.py --call '{"tool":"habit_create","arguments":{"title":"Название привычки","color":"подходящий_цвет","status":0}}'
```

b) Создать конкретную стартовую задачу ("Первый шаг к привычке"):
```bash
python cli.py --call '{"tool":"task_create","arguments":{"title":"Глагол + первый шаг к привычке","start":"2026-02-26"}}'
```

**6. Если исходная задача существовала:**
Отменить оригинал (он заменён проектом):
```bash
python cli.py --call '{"tool":"task_update","arguments":{"id":"T-task-uuid","checked":2}}'
```
В описании или заметке оригинала — упомянуть созданный проект.

### Показать результат

```
Декомпозиция завершена:
- Тип: Проект
- Создан проект "Название" (N задач, M секций)
- MVP: [описание]
- Точка входа: "Первая задача" (на сегодня)
```

## Режим 4: Фокус

**Слоган:** "Меньше открытых — больше закрытых"

### Когда активировать

Пользователь говорит: "что делать сейчас", "фокус", "на чём сфокусироваться", "слишком много задач", "не могу выбрать", "есть N минут", "бракованный день", "паралич выбора".

### Алгоритм

**1. Загрузить задачи:**
```bash
# Задачи на сегодня
python cli.py --call '{"tool":"task_list","arguments":{"start_date_from":"2026-02-26","start_date_to":"2026-02-26","max_count":100}}'
```

| Контекст | Что предложить | Сколько задач | Принцип |
|----------|---------------|---------------|---------|

- "Ублажители": задачи из проектов, связанных с целями/ростом (НЕ рутина, НЕ админщина)
- Микрозадачи: задачи с тегом `micro`, или оценённые <15 мин по формулировке
- Для "плохого дня": самые простые, с конкретным глаголом, не требующие мышления

```
2. [Задача 2] — 20 мин

Всё остальное — потом. Сейчас только это.
```

### Подрежим "Доза мелкой фигни"

Активировать, если накопилось >10 мелких задач (определить по формулировкам: короткие, с простыми глаголами).

**Алгоритм:**
1. Выбрать 5 самых мелких задач
2. Предложить 20-минутный спринт: "Вот 5 мелких задач, пройди одну за другой. Говори номер или 'готово' — отмечу выполненной."
3. По мере выполнения: `task_update` с `checked: 1`
4. В конце: "Спринт завершён! Выполнено N из 5."

### Подрежим "Помодоро"

При запуске фокуса можно предложить "помидор" — 25 минут полной концентрации на одной задаче.

**Алгоритм:**
1. Выбрать одну задачу
2. "Стартуем помидор на 25 минут. Задача: [название]. Полная концентрация."
3. По завершении:
   - Задача выполнена → `task_update` с `checked: 1`. "Отлично! Что дальше?"
   - Не выполнена → "Что помешало? Переформулируем?" → `task_update` с новым `title`
4. Записать время:

Время начала (`start`) — в ISO 8601 с GMT+3. `secondsPassed` = 1500 (25 мин). `source: 0` = ручной ввод.

```bash
python cli.py --call '{"tool":"time_stat_create","arguments":{"start":"2026-02-26T14:00:00.000+03:00","secondsPassed":1500,"relatedTaskId":"T-task-uuid","source":0}}'
```

## Режим 5: Инкубатор (Шкаф-Балкон-Дача)

**Слоган:** "Не сейчас — не значит никогда"

### Когда активировать

Пользователь говорит: "инкубатор", "не сейчас", "когда-нибудь", "может быть потом", "отложить идею", "шкаф", "балкон", "дача", "пересмотреть отложенное".

### Инициализация

При первом использовании создать 3 проекта инкубатора (см. раздел "Инициализация" выше). Проверить наличие через `find_project`:
```bash
python cli.py --call '{"tool":"find_project","arguments":{"name":"Инкубатор: Шкаф"}}'
```
Если `found: false` — создать. Если `found: true` — использовать существующий.

### Три уровня инкубатора

| Уровень | Проект | Обзор | Для чего |
|---------|--------|-------|----------|
| Шкаф | Инкубатор: Шкаф | Еженедельно | "На этой неделе не до этого, но скоро" |
| Балкон | Инкубатор: Балкон | Ежемесячно | "Может через пару месяцев" |
| Дача | Инкубатор: Дача | Ежеквартально | "Когда-нибудь / может быть" |

### Добавление идеи

По умолчанию — в Шкаф (если пользователь не указал уровень явно).

**1. Переформулировать идею в конкретное первое действие для проверки интереса:**

| Идея пользователя | Переформулировать в |
|-------------------|-------------------|
| "Выучить японский" | "Найти один бесплатный курс японского и пройти первый урок" |
| "Начать бегать" | "Найти программу Couch to 5K и пробежать первые 5 минут" |
| "Написать книгу" | "Выделить 15 минут и написать план из 5 глав" |
| "Переехать в другую страну" | "Загуглить требования для ВНЖ в 3 интересных странах" |

**2. Создать задачу:**
```bash
python cli.py --call '{"tool":"task_create","arguments":{"title":"Переформулированная задача","projectId":"P-incubator-shkaf-uuid","deferred":true,"tags":["incubator-tag-uuid"]}}'
```

### Обзор инкубатора

Активировать при "пересмотреть отложенное" или как часть еженедельного обзора (Режим 2A).

**Алгоритм:**

1. Загрузить задачи из каждого уровня:
```bash
python cli.py --call '{"tool":"task_list","arguments":{"project_id":"P-shkaf-uuid","max_count":100}}'
python cli.py --call '{"tool":"task_list","arguments":{"project_id":"P-balkon-uuid","max_count":100}}'
python cli.py --call '{"tool":"task_list","arguments":{"project_id":"P-dacha-uuid","max_count":100}}'
```

2. По каждой задаче — показать и спросить:

| Действие | Что делать |
|----------|-----------|
| **Активировать** | `task_update` с `deferred: false`, `start: сегодня`, перенести в целевой проект (`projectId: P-target`) |
| **Оставить** | Без изменений |
| **Понизить** | Перенести из Шкафа на Балкон или из Балкона на Дачу (`task_update` с `projectId: P-next-level`) |
| **Удалить** | `task_update` с `checked: 2` (cancelled) |

3. **Правила миграции между уровнями:**
   - Идея в Шкафу не заинтересовала 3+ обзора (3+ недели) → предложить Балкон
   - Идея на Балконе 3+ месяца без интереса → предложить Дачу или удаление
   - Идея на Даче 6+ месяцев → предложить удаление ("Если за полгода не потянуло — скорее всего, не потянет")

4. Показать сводку: "Инкубатор: Шкаф — N идей, Балкон — M идей, Дача — K идей. Активировано: X, удалено: Y."

## Общие принципы Дорофеева (справочник)

Краткая шпаргалка по ключевым концепциям книги. Использовать при всех режимах.

### 4 критерия правильной задачи

| # | Критерий | Проверка | Пример нарушения → Исправление |
|---|----------|----------|-------------------------------|
| 1 | Начинается с глагола | Есть ли глагол действия в начале? | "Стоматолог" → "Позвонить и записаться к стоматологу" |
| 2 | "Что конкретно сделать?" | Понятно ли, что именно сделать? | "Отчёт" → "Заполнить раздел 1 шаблона отчёта" |
| 3 | Минимум мышления | Можно начать без подготовки? | "Улучшить презентацию" → "Заменить скриншоты на слайдах 3-5" |
| 4 | Первоочередное действие | Это первый шаг, а не конечная цель? | "Вылечить зуб" → "Позвонить в поликлинику" |

### Мыслетопливо — ограниченный ресурс

- **Сложные задачи — на утро.** Мыслетопливо максимально после сна.
- **Каждое "я запомню" — утечка.** Незаписанная задача висит в фоне и жрёт ресурс.
- **Задача без изменений >24ч — переформулировать.** Наступает "баннерная слепота" — мозг перестаёт видеть задачу.
- **Переключение контекста — дорого.** Каждое переключение = потеря 15-25 минут мыслетоплива.
- **Решения = расход.** Каждое решение тратит мыслетопливо. Минимизировать количество решений в течение дня.

### Задача >30 мин = проект

Если задачу нельзя выполнить за один присест (30 минут) — это проект. Нужна разбивка на шаги по 15-30 минут.

### Техника "Чтобы что?"

Рекурсивный вопрос для проверки актуальности задачи:
- "Зачем мне эта задача?"
- "Чтобы что?"
- "А это чтобы что?"

Если после 3 итераций нет ясного ответа — задачу можно удалить или отправить в инкубатор.

### WIP-лимит

Не больше 3-5 задач "в работе" одновременно. **Чтобы закончить раньше — начни позже.** Много начатого = ничего завершённого. Фокус на завершении, не на начинании.

### Концепция пружины

**Утром — "ублажители" (задачи к целям), не рутину.** За рутину мозг не получает награды. Утреннее мыслетопливо — самое ценное. Тратить его на важное, а не на "разгрести почту".

### "Любить себя в будущем"

Не откладывать мутное. Переформулировать задачу сейчас = подарок будущему себе. "Будущий я" в 11 вечера не разберётся в формулировке "разобраться с тем, что говорил Вася".

### Обезьянка и рациональный менеджер

- **Обезьянка** — часть мозга, которая выполняет задачи. Любит: конкретное, простое, с наградой. Боится: мутного, сложного, непонятного.
- **Рациональный менеджер** — часть мозга, которая планирует. Его задача — подготовить задачи так, чтобы обезьянке было легко.
- **Хорошая формулировка = задача, которую обезьянка ХОЧЕТ делать.**

### Кислородная маска

Принцип из авиации: сначала надень маску на себя, потом помогай другим. В продуктивности: если не заботишься о себе (сон, здоровье, отдых) — всё остальное теряет смысл. В каждом обзоре проверять наличие личных задач.

### Доза мелкой фигни

Когда нет сил на важное — сделать 3-5 мелких задач подряд. Это даёт чувство завершённости и "раскачивает" мозг. Лучше сделать 5 мелочей, чем 0 важных дел.

### Бракованный день

День, когда всё пошло не так. Не пытаться "наверстать" — это путь к выгоранию. Вместо этого: 1-2 мелкие задачи для чувства завершённости + отдых. Завтра будет лучше.

## Маршрутизация между режимами

Режимы связаны между собой. Вот когда переключаться:

| Текущий режим | Событие | Перейти в |
|--------------|---------|-----------|
| Режим 1 (Разгрузка) | Элемент >30 мин | Режим 3 (Разбивка) |
| Режим 1 (Разгрузка) | Идея "на потом" | Режим 5 (Инкубатор) |
| Режим 2A (Еженедельный обзор) | Нашли отложенные идеи | Режим 5 (Обзор инкубатора) |
| Режим 2B (Вечерний обзор) | Завершён | Режим 8 (Итоги дня) |
| Режим 3 (Разбивка) | Проект создан, что дальше? | Режим 4 (Фокус) |
| Режим 4 (Фокус) | Слишком много мелочи | Подрежим "Доза мелкой фигни" |
| Режим 7 (Триггеры) | Элемент из категории | Режим 1 (Разгрузка, для обработки) |
| Режим 8 (Итоги дня) | Чек-лист заполнен | Режим 2B (Вечерний обзор) |

**Переходы — только по согласию пользователя.** Предложить, но не навязывать.

## Технические заметки

### Формат дат

- Дата без времени: `"2026-02-26"` (для `start`, `deadline`)
- Дата со временем: `"2026-02-26T10:00:00.000+03:00"` (GMT+3, для встреч с `useTime: true`)
- Для `habit_progress_create`: `"2026-02-26"` (только дата)

### Формат тегов

При `task_create` передавать массив UUID тегов: `"tags": ["tag-uuid-1", "tag-uuid-2"]`. UUID получать из кэша `references/tags.json` или через `find_tag`.

### Состояния задач

| Поле | Значение | Смысл |
|------|----------|-------|
| `checked: 0` | Не завершена | Активная задача |
| `checked: 1` | Завершена | Выполнено |
| `checked: 2` | Отменена | Cancelled |
| `state: 0` | Pinned | Закреплена (фокус) |
| `deferred: true` | Отложена | Не показывать в основном списке |

### Заметки (Note в Delta-формате)

Для создания заметок к задачам/проектам:
```bash
python cli.py --call '{"tool":"note_create","arguments":{"containerId":"T-task-uuid","content":[{"insert":"Текст заметки\n"}],"contentType":"delta"}}'
```
Последний `insert` должен заканчиваться на `\n`.

### Уведомления

```json
{"notifies": [60, 15], "notify": 1}
```
`notifies` — массив минут до события. `notify: 1` — включить уведомления.

### Рекуррентные задачи

Для повторяющихся задач использовать поле `recurrence`. Формат зависит от API (см. документацию Singularity).

### Кэш и производительность

- **Всегда читать кэш** (`references/projects.json`, `references/tags.json`) вместо API-вызовов для поиска проектов/тегов
- **После создания** проекта/тега — вызвать `rebuild_references` для обновления кэша
- **task_list** — единственный способ получить задачи (в кэше задач нет)
- **max_count: 1000** — максимум для одного запроса task_list
